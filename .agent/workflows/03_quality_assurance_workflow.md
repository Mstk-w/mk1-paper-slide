# 品質保証ワークフロー (Workflow: Quality Assurance)

スライド生成の各段階で確認すべき品質基準です。

1. **コンテンツ品質チェック (Logic QA)**
    - **タイミング**: スライド構成案（JSON）作成時
    - **チェック項目**:
        - [ ] 「だ・である」調で統一されているか？
        - [ ] 曖昧なカタカナ語や感情的な表現が含まれていないか？
        - [ ] 8つのボックス（背景→課題→計画→効果）の論理は繋がっているか？
        - [ ] 予算や数値目標は具体的か？（不明な場合は仮置きされているか？）

2. **デザイン・実装品質チェック (Design QA)**
    - **タイミング**: generate_slide.py 実行後
    - **チェック項目**:
        - [ ] A3横の比率で生成されているか？
        - [ ] 日本語フォント（Meiryo等）が指定されているか？
        - [ ] 文字がテキストボックスからはみ出していないか？（※AIは視覚確認できないため、文字数ベースで警告を行う）
        - [ ] 枠線や配置がズレていないか？

3. **コード品質チェック (Code QA)**
    - **タイミング**: スクリプト修正時
    - **チェック項目**:
        - [ ] 型ヒントは付与されているか？
        - [ ] 定数はハードコーディングされず、定義されているか？
        - [ ] 重要なロジックには日本語コメントがあるか？
