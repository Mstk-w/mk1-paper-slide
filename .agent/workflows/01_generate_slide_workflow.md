# スライド生成ワークフロー (Web App Workflow)

ブラウザ上のWebインターフェース（Streamlit）を通じた生成プロセスです。

1. **テーマ入力 (Input Phase)**
    - ユーザーがブラウザ画面で「テーマ」と「作成部局」を入力する。

2. **壁打ち・ヒアリング (Interactive Heating)**
    - AIが .agent/rules/02_government_logic.md に基づき、不足情報をチャット形式で質問する。
    - ユーザー回答を受け、AIが8ボックス構成の内容を推論で埋める。
    - 「現状のデータはありますか？」「具体的な数値目標は？」等を深掘りする。

3. **構成案の提示・修正 (Drafting Phase)**
    - AIがJSON形式のスライド構成案を作成し、画面上に構造化して表示する。
    - ユーザーは各セクション（例：課題、解決策）の内容を確認し、必要であればチャットで「〇〇の部分をもう少し具体的にして」と指示する。
    - AIは構成案を修正し、最新のJSONをプレビューする。

4. **スライド生成実行 (Generation Phase)**
    - ユーザーが「生成」ボタンを押下する。
    - バックエンドで generate_slide.py が実行される。
    - **動的レイアウト**: 値が空（
ull または空文字）のボックスは自動的に削除され、レイアウトが調整される。

5. **ダウンロード・確認 (Delivery)**
    - 生成完了後、ダウンロードボタンが表示される。
    - ユーザーは .pptx をダウンロードし、微調整を行う。
